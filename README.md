# 岩芯样本切片实验室

运行：

```bash
npm start
```

访问`http://localhost:3025`。支持样本创建、切片任务、步骤记录和交付统计。

## CSV批量导入

系统支持通过CSV文件批量导入岩芯样本和切片任务。

### 下载模板

在导入面板中点击「📥 下载CSV模板」按钮可一键下载系统预置的CSV模板文件，模板包含完整的列名和示例数据。

### 字段说明

| 列名 | 说明 | 必填 | 示例 |
|------|------|------|------|
| **样本编号** | 已有样本的编号，用于追加切片到该样本。留空则创建新样本。 | 否 | CORE-001 |
| **项目** | 项目名称，相同项目、钻孔、箱号、深度、负责人的行会自动合并为一个样本。 | 是 | 东岭铜矿薄片 |
| **钻孔编号** | 钻孔编号。 | 是 | ZK-17 |
| **岩芯箱号** | 岩芯箱编号。 | 是 | BX-09 |
| **取样深度** | 取样深度范围，格式为「起始-结束m」。 | 是 | 128.4-128.8m |
| **负责人** | 样本负责人姓名。 | 是 | 陆川 |
| **切片编号** | 切片编号，格式为「SL-数字-字母」，全站唯一。 | 是 | SL-011-A |
| **染色方法** | 制片方法，如普通薄片、茜素红染色、光片等。 | 是 | 普通薄片 |

### 导入方式

#### 1. 创建新样本

- **样本编号**列留空
- 系统会自动为新样本生成编号（格式：CORE-时间戳）
- 相同「项目+钻孔+箱号+深度+负责人」的多行数据会自动合并为一个样本，每行对应一个切片

#### 2. 追加切片到已有样本

- 在**样本编号**列填写已存在的样本编号（如 CORE-001）
- 系统会将该行的切片任务追加到指定的现有样本中
- 样本的其他信息（项目、钻孔等）以已有样本为准，CSV中的信息会被忽略
- 支持一次性为多个已有样本追加切片

### 格式要求

- 文件编码：UTF-8（推荐带BOM，兼容Excel）
- 切片编号格式：`SL-\d+-[A-Za-z]+`，例如 SL-011-A、SL-011-B
- 深度格式：`数字-数字m`，例如 128.4-128.8m
- 每行代表一个切片任务
- 支持中文或英文列名（系统自动识别）

### 支持的英文列名别名

| 中文列名 | 支持的英文别名 |
|---------|--------------|
| 样本编号 | sampleId, sample_id |
| 项目 | project |
| 钻孔编号 | borehole, 钻孔 |
| 岩芯箱号 | coreBox, core_box, 箱号 |
| 取样深度 | depth, 深度 |
| 负责人 | owner |
| 切片编号 | sliceId, slice_id |
| 染色方法 | method, 方法, 制片方法 |

## 样本筛选功能

页面右侧顶部提供了**样本筛选**面板，支持以下筛选条件：

| 筛选条件 | 说明 |
|---------|------|
| **项目** | 按项目名称筛选，下拉列表自动提取所有已存在的项目 |
| **钻孔编号** | 按钻孔编号（如 ZK-17）筛选 |
| **岩芯箱号** | 按岩芯箱号（如 BX-09）筛选 |
| **负责人** | 按负责人姓名筛选 |
| **样本状态** | 按制片状态筛选：待切割、制片中、待观察、已交付 |
| **交付状态** | 按交付状态筛选：未交付、已交付 |

### 使用方式

1. **设置筛选条件**：在筛选面板的任意下拉框中选择目标值，列表和统计数字会**实时更新**
2. **组合筛选**：可以同时设置多个筛选条件，条件之间为「且」的关系
3. **清除筛选**：点击「清除筛选」按钮一键重置所有筛选条件
4. **筛选持久化**：筛选条件会自动保存在 URL 查询参数中，刷新页面后会自动恢复最近一次筛选条件，也可以复制 URL 共享筛选结果

### 数据统计

- 顶部 4 个统计卡片（待切割、制片中、待观察、已交付）会根据筛选结果同步更新
- 统计卡片下方显示筛选后的样本总数

### 空结果提示

当没有符合筛选条件的样本时，会显示友好的空状态提示，可调整筛选条件后重试。
